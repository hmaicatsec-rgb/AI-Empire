# AI Empire - Engineering Rules

> Version: 1.0
>
> Mission:
>
> Become a world-class AI Engineer and build an AI company capable of creating autonomous AI agents, software platforms, cybersecurity systems, operating systems, and complete AI ecosystems.

---

# Rule 1 - Understand Before Building

Always understand WHY before HOW.

Never blindly copy code.

---

# Rule 2 - One Step at a Time

Focus on a single task.

Finish it completely before starting the next one.

---

# Rule 3 - Think Like an Engineer

Always ask:

- Why?
- Is this scalable?
- Is this maintainable?
- Is this secure?
- Is there a better architecture?

---

# Rule 4 - Keep Everything Organized

Development Workspace:

D:\Development

Projects:

D:\Development\AI\projects

Models:

D:\Development\AI\Ollama\Models

Never scatter files across the system.

---

# Rule 5 - Git First

Every serious project must use Git.

Commit meaningful progress.

Write clear commit messages.

---

# Rule 6 - Documentation Matters

Always document:

- Decisions
- Architecture
- Folder structure
- Lessons learned

Good documentation saves hundreds of hours.

---

# Rule 7 - AI is a Team

Each AI has a different responsibility.

ChatGPT

- Mentor
- AI Architect
- Teacher
- Planning

Claude Code

- Software Engineering
- Refactoring
- Large Codebases

Gemini

- Reasoning
- System Design
- Cybersecurity
- AI Architecture

Gemma

- Quick Tasks
- Notes
- Summaries

---

# Rule 8 - Choose the Right AI

Never use the same model for every task.

Always choose the best tool for the job.

---

# Rule 9 - Security First

Never expose:

- API Keys
- Passwords
- Tokens
- Secrets

Use environment variables whenever possible.

---

# Rule 10 - Build for the Future

Every project should be scalable.

Avoid shortcuts.

Design systems that can grow.

---

# Rule 11 - Learn Deeply

Do not memorize.

Understand concepts.

Master fundamentals.

---

# Rule 12 - Professional Mindset

Always write:

- Clean code
- Readable code
- Maintainable code

Not just working code.
